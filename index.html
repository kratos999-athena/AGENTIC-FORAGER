How It Works

Four agents. One pipeline. Zero hype.

Each agent in the pipeline has a single, auditable responsibility. No agent invents data — every claim traces to a source. The entire execution trace is surfaced in the UI so analysts can inspect every decision.

Agent 1
The Scout
Signal Extraction

Operating continuously, The Scout monitors raw data streams to separate genuine engineering milestones from marketing noise. Utilising structured JSON classification and an auditable Chain-of-Thought (CoT), it evaluates architectural shifts and developer adoption velocity to emit high-confidence signals downstream.

Agent 2
The Quant
Deterministic Metrics

Taking the baton from The Scout, The Quant operates purely on hard data with zero LLM inference. It autonomously executes GraphQL queries against GitHub to extract 30/60/90-day commit velocity, issue resolution timelines, and repository trajectory, explicitly flagging data gaps to ensure analytical transparency.

Agent 3
The Partner
Grounded Synthesis

The Partner synthesizes the qualitative narrative and quantitative metrics into a structured investment memo. Constrained by a strict hallucination firewall, it is forced to deliberate internally before citing exact metrics from The Quant's payload, producing a mathematically grounded thesis.

Agent 4
The Assistant
Interactive Due Diligence

A real-time, conversational interface allowing analysts to interrogate the memo. The Assistant utilises a dedicated <think> reasoning trace to verify data points privately before answering. Every interaction is shielded by a semantic Llama Guard firewall to prevent prompt injection and data exfiltration.

Built for rigour, not speed.

  • Every LLM output is validated against a Pydantic schema before it propagates downstream
  • JSON-mode enforced on all structured calls — no regex fallbacks on production paths
  • GitHub metrics are fetched via authenticated GraphQL, not scraped HTML
  • Llama Guard 4 screens every user message before it reaches Agent 4
  • All API keys managed through a pool with per-key rate-limit tracking and automatic failover

Transparent by design.

  • Full execution trace available on every card — inspect every agent decision
  • Chain-of-thought fields captured and displayed for Agents 1 and 3
  • Data gaps explicitly flagged — never silently inferred
  • Red-team debate transcript persisted alongside the memo for post-hoc review
  • Researcher auditor programme adds human-in-the-loop verification layer
